# File Guide

Quick reference for every file touched during the BCB330 project.

---

## API Models

**`api/models/efp_schemas.py`**
Defines the schema for all 191 eFP databases as a compact list of `(name, species)` tuples plus
two lookup sets (`_PROBESET_DBS`, `_UTF8MB4`); a single builder function expands each tuple into
a full SQLAlchemy-compatible schema dict at import time.

**`api/models/efp_dynamic.py`**
Reads `SIMPLE_EFP_DATABASE_SCHEMAS` and generates one SQLAlchemy ORM model class per database at
startup, stored in `SIMPLE_EFP_SAMPLE_MODELS`; enables a single query path for all 191 databases
without writing a class per database.

---

## API Services

**`api/services/efp_data.py`**
Central query service for all eFP databases; resolves the right engine (MySQL bind → SQLite mirror
→ in-memory seed data), runs a parameterised `SELECT` against `sample_data`, and returns a
uniform result dict used by all eFP resource endpoints.

**`api/services/efp_bootstrap.py`**
Builds the SQLite mirror databases used in CI and local development by reading the SQL seed files
in `config/databases/` and writing `.db` files to a temp directory on startup.

---

## API Resources

**`api/resources/gene_expression.py`**
`GET /gene_expression/expression/<database>/<gene_id>` — validates the gene ID against its
species, normalises it (e.g. strips maize transcript suffix), converts to a probeset if needed,
then delegates to `efp_data.query_efp_database_dynamic`.

---

## API Utilities

**`api/utils/gene_id_utils.py`**
Species detection, gene ID validation, normalisation (maize `_T##` suffix, barley `.V3` → `.1`),
and probeset conversion; `PROBESET_DATABASES` lists every database that requires a gene→probeset
lookup before querying; only Arabidopsis conversion is fully implemented.

---

## Scripts

**`scripts/benchmark_efp.py`**
Times the flat vs dynamic schema approaches for model creation and query latency; results are
summarised in `archive/notes/note_benchmarking_results.md`.

**`scripts/bootstrap_simple_efp_dbs.py`**
One-off script to generate the SQLite seed files in `config/databases/` from the BAR MySQL
server; run this when adding a new database to the schema catalog.

**`scripts/proofread_efp_values.py`**
Spot-checks expression values returned by the API against known reference values; used to verify
that the dynamic query path returns the same results as the legacy CGI.

---

## Data Files

**`data/efp_info/efp_species_view_info.json`**
Scraped from the BAR eFP XML files by `archive/dev_scripts/scrape_species_view_info.py`; maps
each species → views → groups/controls/tissues/samples; used by the samples endpoint.

**`data/efp_info/efp_species_view_info_typed.json`**
Same structure as above with an added `data_type` field (`"Microarray"`, `"RNA-Seq"`, or
`"Unknown"`) on each view; generated by `archive/dev_scripts/annotate_efp_data_types.py`.

---

## Tests

**`tests/models/test_efp_schemas.py`**
Verifies that all 191 schema dicts have the required keys, correct column types (VARCHAR 255),
valid charsets (`latin1` / `utf8mb4`), and correct species metadata.

**`tests/models/test_efp_dynamic.py`**
Verifies that all 191 ORM models are generated correctly with the right column types, primary
keys, class names, and `__bind_key__` attributes.
